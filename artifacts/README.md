# Artifacts

This directory holds small, checked-in validation evidence and historical
milestone summaries. It should not become a dumping ground for opaque outputs.

## Rules

Keep an artifact in git only when it is:

- small,
- reproducible or historically useful,
- safe for public version control,
- documented in the inventory below.

Do not commit PHI, patient-level raw extracts, restricted datasets, large raw
exports, or scratch files that cannot be regenerated or explained.

## Regeneration policy

- Prefer `uv run ...` commands for new or refreshed artifacts.
- Update this inventory in the same commit as any artifact addition, removal, or
  regeneration.
- If an artifact becomes obsolete but still useful for history, mark it
  `deprecated` instead of silently deleting it.
