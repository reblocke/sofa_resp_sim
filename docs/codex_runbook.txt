Implementation Ticket: Python Simulation of Respiratory SOFA Scoring (SQL-Parity)
Objective

Build a Python simulation framework that:

Simulates SpO2 observations over time, plus a categorical oxygen / ventilatory support variable (and any other minimal derived fields needed).

Computes the respiratory SOFA component exactly as implemented in the current SQL, specifically the logic embodied by:

SOFA Respiratory Detail.sql (event-level PaO2/FiO2 and respiratory SOFA temp score derivation)

Trops Measures - SOFA.sql (acute vs baseline selection rules and final score selection/adjustments for SOFA_PULM and SOFA_PULM_BL)

Allows parameterized simulation experiments (observation frequency/length, oxygen escalation thresholds, measurement noise, etc.) and outputs distributions of resulting respiratory SOFA scores (and intermediate artifacts like PF ratios and count-of-qualifying PF records).

Non-goal: Do not implement “clinically correct” SOFA. Implement the SQL behavior, including its quirks and gating rules.

Scope Boundaries
In scope

Respiratory component only:

Event-level PF ratio calculation and temp respiratory score (“SOFA_RESP_TEMP” behavior).

Acute-initial (“SOFA_PULM”) and baseline (“SOFA_PULM_BL”) respiratory component selection rules as used in the Trops measures query.

Optional: delta respiratory component (max(index − baseline, 0)) if you want it for later analyses; keep this modular.

Out of scope

Other SOFA components (CV, renal, etc.).

Reconstructing EHR-derived support episodes from free-text fields (tube activity, vent mode, etc.). In simulation, support is an input.

Required SQL-Parity Rules to Implement
A. SpO2 → PaO2 conversion (Ellis equation proxy)

From SOFA Respiratory Detail.sql, CTE convert_spo2_to_pao2:

Compute pao2_calc only when 50 ≤ SpO2 ≤ 96 (inclusive).

Otherwise pao2_calc = NULL.

The expression can be implemented in simplified mathematical form (equivalent to the SQL):

Let s = SpO2 / 100.

Let x = 11700 / ((1/s) − 1).

Let y = sqrt(50^3 + x^2).

Then pao2_calc = round( (y + x)^(1/3) − (y − x)^(1/3), 1 ).

Rounding requirement: SQL is Oracle-style ROUND(value, n); do not use Python’s default bankers rounding. Implement an Oracle-compatible rounding helper (half-up for positive values is the usual behavior needed here).

B. PaO2 selection priority

From combine_resp_events:

pao2 = NVL(pao2_meas, pao2_calc)

pao2_priority = 1 if measured PaO2 exists, else 2 if derived PaO2 exists, else NULL.

For your initial simulation (SpO2-only), pao2_meas will typically be absent, so pao2_priority will usually be 2 when SpO2 is in-range.

C. FiO2 “prioritized” value and FiO2 priority

From add_fio2_priority:

Compute fio2_prioritized (percent, 21–100) and fio2_priority using these precedence rules:

If room air indicator is true (oxygen_therapy contains “room air” in SQL):

fio2_prioritized = 21

fio2_priority = 1

Else if invasively ventilated (invasive_ind = 1):

Prefer fio2_set, then fio2_meas, then fio2_abg

Priority mapping: set=2, meas=3, abg=4

Else (not invasive):

Prefer fio2_meas, then fio2_set, then fio2_abg

Priority mapping: meas=2, set=3, abg=4

Else if oxygen_flow_rate between 0 and 15:

fio2_prioritized = oxygen_flow_rate * 3 + 21

fio2_priority = 5

Simulation implication: You will need a consistent mapping from your simulated support category to one or more of:

room air flag

oxygen flow rate (0–15 for low-flow behavior)

fio2_set / fio2_meas (for HFNC/NIPPV/IMV use cases)

D. FiO2 temporal association windows

From lookback_for_fio2 and pick_fio2:

Per record, compute:

fio2_day_lookback: last non-null fio2_prioritized in the prior 24 hours, including current timestamp.

fio2_minute_lookback: last non-null fio2_prioritized in [t − 14 minutes, t − 1 minute].

fio2_minute_look_fw: first non-null fio2_prioritized in [t + 0 minutes, t + 5 minutes] (includes the current record).

Then compute:

If room air: fio2_lookback = fio2_prioritized, fio2_lookback_priority = fio2_priority

Else:
fio2_lookback = COALESCE(fio2_minute_lookback, fio2_minute_look_fw, fio2_day_lookback)
and similarly for fio2_lookback_priority.

Partitioning constraint (SQL): These lookbacks are computed within partitions that include invasive_ind. In simulation, reproduce the intent: do not carry FiO2 across an invasive/non-invasive boundary when computing these lookbacks.

E. PF ratio computation (PF_RATIO_TEMP)

From assign_pf_ratio_temp:

Compute pf_ratio_temp with the following case logic (in order):

If pao2_priority = 2 and spo2 IN (99, 100) and pao2 / fio2_lookback <= 4 * altitude_factor then pf_ratio_temp = NULL.

Note: with the SpO2≤96 restriction, this branch should be unreachable in SpO2-derived cases, but implement it anyway for strict parity.

Else if fio2_lookback exists and is not zero:

pf_ratio_temp = round( pao2 / (fio2_lookback / 100), 2 )

Else if any fio2_prioritized exists in the same (sofa_ts, quartile) block:

pf_ratio_temp = NULL
(This prevents defaulting to room air when a better FiO2 exists nearby in that block.)

Else if pao2_priority = 2 and still no FiO2:

pf_ratio_temp = round( pao2 / 0.21, 2 )

Else:

pf_ratio_temp = NULL

Key: The quartile concept is used here (see next section). Even if your simulation always provides FiO2, implement the quartile logic so the scoring engine remains SQL-faithful.

F. Rubric mapping PF ratio → respiratory SOFA temp score

From sofa_resp_rubric and the rubric join in add_sofa_resp_score_temp:

Rubric bins (lower bound inclusive, upper bound exclusive), after altitude adjustment:

0 ≤ PF < 100 → score 4

100 ≤ PF < 200 → score 3

200 ≤ PF < 300 → score 2

300 ≤ PF < 400 → score 1

PF ≥ 400 → score 0

In SQL, the join applies altitude_factor to the bin edges:

Compare pf_ratio_temp to pf_ratio_min * altitude_factor and < pf_ratio_max * altitude_factor.

Implement this exactly.

G. Respiratory temp score deprecation in resp detail

From add_sofa_resp_score_temp:

If not on invasive or non-invasive support at that timestamp (NVL(invasive_ind, support_ind) is null) AND rubric score is 3 or 4:

set sofa_resp_temp = 2

Else sofa_resp_temp = rubric score

H. Trops Measures respiratory rules: acute vs baseline selection and gating

From Trops Measures - SOFA.sql, CTE sofa_detail_combined + downstream logic:

Acute respiratory records included in scoring

Include a resp record for the acute window only if:

pf_ratio_temp IS NOT NULL

COALESCE(fio2_minute_lookback, fio2_minute_look_fw) IS NOT NULL
(FiO2 within the -14 to -1 lookback OR 0 to +5 look-forward)

etime_ts >= admit_dts + acute_begin where acute_begin = -0.25 days (−6 hours)

Baseline respiratory records included in scoring

Include baseline resp records if:

pf_ratio_temp IS NOT NULL

etime_ts < admit_dts + acute_begin
(no requirement on minute lookback/forward)

Additional support-based cap applied again in Measures query

Measures query sets:

sofa_temp = 2 if support_type NOT IN ('IMV', 'NIPPV') and sofa_resp_temp >= 3

Else sofa_temp = NVL(sofa_resp_temp, 0)

Important nuance: This is stricter than the resp-detail cap because it does not treat HFNC as qualifying support for scores 3–4.

Acute-only “single PF ratio” suppression rule

From adjust_component_scores:

If in acute period and:

etype_cd = 'PFRATIO'

count_pf_ratio = 1 (within acute window)

resp_support_ind = 0 (where resp_support_ind = 1 only if support_type IN ('IMV','NIPPV'))

sofa_temp > 0

Then set sofa_temp2 = 0.

This is a major driver of outcomes in simulation: if you only generate one qualifying PF ratio in acute and you’re not on IMV/NIPPV, the final acute score is forced to 0.

Final acute SOFA_PULM selection

Pick the single acute respiratory record (within acute window) that ranks highest by:

sofa_temp2 descending

resp_support_ind descending

etime_ts ascending

The selected record’s sofa_temp2 becomes the final SOFA_PULM. If no qualifying records exist, it defaults to 0.

Final baseline SOFA_PULM_BL selection

Within the baseline eval window defined as:

baseline_begin = add_months(trunc(admit_dts), -36)

baseline_end = trunc(admit_dts) - 7 days

Pick by:

sofa_ts descending (closest baseline day to index)

sofa_temp2 descending

etime_ts descending

Then suffix to SOFA_PULM_BL. If no qualifying baseline records exist, it defaults to 0.

Data Model for the Scoring Engine
Input table (one encounter simulation)

Represent observations as a time-ordered table with at least:

etime_ts (datetime)

spo2_obs (numeric, typically 0–100)

Support fields:

support_type ∈ {IMV, NIPPV, HFNC, OSA, SURG IMV, None}
(For low-flow oxygen or room air, use None; do not invent new support_type values unless you also map them.)

invasive_ind boolean (derive from support_type == IMV/SURG IMV)

support_ind boolean (derive true for HFNC or NIPPV if you want resp-detail parity)

FiO2 source fields (at least one pathway must exist for non-room-air):

fio2_set (percent)

fio2_meas (percent)

fio2_abg (percent)

oxygen_flow_rate (L/min, used only when 0–15)

is_room_air boolean

Encounter anchors:

admit_dts datetime (index time for acute/baseline logic)

altitude_factor (default 0.85 unless explicitly varied)

Output table (event-level)

Return a table with all intermediate columns needed for debugging/analysis, at minimum:

pao2_calc, pao2, pao2_priority

fio2_prioritized, fio2_priority

fio2_minute_lookback, fio2_minute_look_fw, fio2_day_lookback, fio2_lookback

pf_ratio_temp

sofa_resp_temp (resp-detail temp)

sofa_temp and sofa_temp2 (post-Measures adjustments)

resp_support_ind

sofa_eval_period, sofa_ts, and quartile (needed for internal logic parity)

Output summary (encounter-level)

SOFA_PULM (acute initial)

SOFA_PULM_BL (baseline)

Optional: SOFA_PULM_DELTA = max(SOFA_PULM − SOFA_PULM_BL, 0)

Diagnostics:

count_pf_ratio_acute (as counted by the Measures logic)

n_qualifying_pf_records_acute and n_qualifying_pf_records_baseline

Simulation Layer Requirements
Goal

Generate a realistic-enough synthetic dataset for repeated experiments while staying aligned with the scoring engine’s required inputs.

Core simulation knobs (parameters)

At minimum support:

Time grid

Acute window length (default should cover −6h to +24h relative to admit)

Baseline window inclusion (optional; if included, must support at least one day between −36 months and −7 days)

Observation frequency (e.g., every 5 min, 15 min, 60 min)

Total number of observations (derived or explicit)

True SpO2 process

Distributional model for spo2_true(t):

e.g., baseline mean + occasional desaturation episodes + autocorrelation

Parameters controlling:

mean level

variance

temporal correlation (e.g., AR(1) coefficient)

probability and depth/duration of desaturation episodes

Measurement noise model

Additive noise (e.g., Gaussian) with configurable SD

Clamp to [0,100]

Rounding rule to mimic EHR storage (integer rounding is common; make this configurable)

Decide whether support decisions are based on true vs observed SpO2 (default to observed)

Oxygen/support policy

A rule that maps SpO2 (typically observed) to oxygen delivery category

Must be parameterized to allow “different thresholds for applying supplemental oxygen”

Output should map to fields used by the scoring engine:

For room air: is_room_air = True

For low-flow: set oxygen_flow_rate in 0–15 and is_room_air = False

For HFNC/NIPPV/IMV: provide support_type and a FiO2 source (fio2_set or fio2_meas)

Provide default policy examples (but keep policy pluggable):

single threshold (room air vs oxygen)

multi-step ladder (room air → low-flow → HFNC → NIPPV → IMV)

FiO2 assignment per category

For low-flow oxygen: sample oxygen_flow_rate and let the SQL rule compute FiO2 as flow*3+21

For HFNC/NIPPV/IMV: provide FiO2 as percent:

either fixed values

or sampled from a distribution (e.g., truncated normal)

Allow scenarios where FiO2 changes occur within <15 minutes to exercise minute lookback behavior.

Altitude factor

Default 0.85

Allow varying it to see threshold shifts

Simulation outputs for analysis

For each parameter set and number of Monte Carlo replicates:

Distribution of final SOFA_PULM and SOFA_PULM_BL

Distribution of count_pf_ratio_acute

Distribution of whether the “single PF ratio suppression” rule triggered

Optional: distribution of PF ratios and derived PaO2 values

Acceptance Criteria
Functional parity

Given a deterministic toy dataset (timestamps, spo2, fio2, support_type, admit time), the Python engine must reproduce:

pao2_calc presence/absence rules (50–96 only)

pf_ratio_temp rules including room-air defaulting behavior only in the allowed case

rubric scores with altitude-adjusted thresholds

Measures-stage capping (only IMV/NIPPV allow 3–4)

acute single-PF suppression rule

final acute selection ordering and baseline selection ordering

Simulation capability

Must be able to run at least:

1,000 replicates in a single parameter configuration

a grid sweep across (observation frequency × noise SD × oxygen threshold)

Output includes a summary table with estimated probabilities for each SOFA_PULM class (0–4).

Reproducibility

Fixed seed reproduces identical results.

Results capture parameters and seed in outputs.
