---
name: pr-draft-summary
description: Use when work is ready for review and you need a concise, high-signal pull request summary with validation evidence and remaining risks.
---

Produce:

1. Title
2. Problem summary
3. What changed
4. Validation performed
5. Docs/artifacts updated
6. Remaining risks or follow-ups

For this repo, always include:
- whether scoring/simulation math changed
- whether app behavior changed
- whether docs or artifacts changed
- exact commands used for validation
